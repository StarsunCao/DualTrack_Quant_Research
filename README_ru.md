# DualTrack Quant Research

**Платформа сравнительных экспериментов многоканальной количественной торговли** — строгое сравнение традиционного машинного обучения (Fitting) и больших языковых моделей (Semantic Reasoning) по показателям ROI, устойчивости и инженерной осуществимости в количественной торговле, особенно в условиях событий «чёрного лебедя».

> **Ключевая позиция**: Данный проект не является стратегией слияния сигналов, а представляет собой **инженерную инфраструктуру (Testbed)** для сравнительных экспериментов, обеспечивающую честную конкуренцию между «обучением» (fitting) и «рассуждением» (reasoning) в одинаковых рыночных условиях.

---

## Обзор

DualTrack — это фреймворк для бэктестинга количественной торговли на Python 3.12+, поддерживающий:

- **3 модели машинного обучения**: Logistic Regression, LSTM, LightGBM
- **6 больших языковых моделей**: DeepSeek-V3.2, V4-Flash, R1-8B, Qwen 3.5-397B, GLM-5.1, Gemma-4-31B, с облачным и локальным развёртыванием
- **SmartPromptAgent**: Улучшенный LLM-агент с техническими индикаторами, историей цен и замкнутой памятью решений
- **2 рынка**: Акции Китая (CSI300) и акции США (QQQ/NASDAQ-100)
- **Независимый бэктестинг каналов**: Каждая модель генерирует сигналы и проводит бэктестинг независимо, без слияния сигналов
- **Многомерная оценка**: Финансовые метрики (Sharpe, MaxDD, WinRate) + инженерные метрики (Latency, Cost)

## Поддерживаемые модели

### ML каналы (машинное обучение)

| Модель | Тип | Особенности |
|--------|-----|-------------|
| `lr` | Логистическая регрессия | Линейный базис, экстремально быстрая |
| `lstm` | LSTM (PyTorch, MPS-ускорение) | Последовательное моделирование, оптимизация для Apple Silicon |
| `lgb` | LightGBM | Ансамблевое обучение, интерпретируемая важность признаков |

### LLM каналы (большие языковые модели)

| Модель | Развёртывание | Тестовые рынки | Описание |
|--------|--------------|-----------------|----------|
| `deepseek-v3.2` | Облако (SiliconFlow) | Акции Китая + США | DeepSeek V3.2 стандартная |
| `deepseek-v4-flash` | Облако (SiliconFlow) | Акции Китая + США | DeepSeek V4 Flash облегчённая |
| `deepseek-r1-8b` | Локально (Ollama) | Акции Китая + США | DeepSeek R1 8B дистиллированная |
| `qwen3.5-397b` | Облако (SiliconFlow) | Акции Китая + США | Qwen 3.5 397B полная версия |
| `glm-5.1` | Облако (SiliconFlow / DashScope) | Акции Китая + США | GLM-5.1 |
| `gemma-4-31b` | Облако (SiliconFlow) | Акции Китая + США | Gemma 4 31B открытая модель |

### SmartPromptAgent (Улучшенный LLM-агент)

SmartPromptAgent переводит LLM-каналы из «скриптинга промптов» в современную парадигму агентов, сохраняя один вызов LLM на торговый день для экономии затрат:

- **Технические индикаторы**: RSI, MACD, полосы Боллинджера, выравнивание MA, соотношение объёма и цены — предварительно вычисляются Python и вводятся в виде текстовых сводок
- **История цен**: Таблица тренда цен за последние 5 дней с отметками объёма
- **Замкнутая память**: Прошлые решения с фактической доходностью рынка, автоматически заполняемой обратно, позволяя LLM видеть, были ли его предыдущие вызовы правильными или ошибочными
- **Нулевое заглядывание в будущее**: Все введённые данные строго ограничены T-1 и ранее, обеспечивается assert-защитой `MarketEnricher`
- **Оптимизация затрат**: ~300 дополнительных токенов в день, по-прежнему 1 вызов LLM/торговый день (против 2-4x для мульти-вызовных агентских дизайнов)
- **Temperature**: 0.1 (низкая случайность для воспроизводимых бэктестов)

```bash
# Запуск бэктестинга SmartPromptAgent
uv run python main.py run_llm_agent_backtest \
  --symbol CSI300 \
  --start 2020-01-02 \
  --end 2024-12-31 \
  --executor siliconflow \
  --model deepseek-ai/DeepSeek-V3.2
```

> **Устройство локального развёртывания**: MacBook Air M2 (24 ГБ ОЗУ + 512 ГБ SSD)
>
> **Примечание о Qwen**: Из-за возможного срабатывания политической цензуры модели Qwen не тестировались на рынке акций Китая, только на рынке США.

## Быстрый старт

### Установка

**Данный проект использует `uv` в качестве менеджера пакетов (pip запрещён)**

```bash
# Установка uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# Клонирование и установка зависимостей
git clone https://github.com/StarsunCao/DualTrack_Quant_Research.git
cd DualTrack_Quant_Research
uv sync
```

### Запуск бэктестинга

```bash
# Бэктестинг одной модели
uv run python main.py run --track lr --symbol CSI300
uv run python main.py run --track lstm --symbol CSI300
uv run python main.py run --track lgb --symbol QQQ

# Бэктестинг LLM (требуется действительный кэш или API)
uv run python main.py run --track deepseek-v3.2 --symbol CSI300
uv run python main.py run --track glm-5.1 --symbol QQQ

# Полное сравнение моделей (требуются готовые кэши всех LLM)
uv run python main.py run --track all --compare --symbol CSI300
```

### Построение кэша LLM

Вызовы LLM API дороги; проект поддерживает автономное кэширование с возможностью возобновления:

```bash
uv run python main.py cache-build \
  --symbol CSI300 \
  --start 2020-01-02 \
  --end 2024-12-31 \
  --news-file data/raw/csi300_news_combined_2020_2024.csv \
  --executor siliconflow \
  --model deepseek-ai/DeepSeek-V3.2 \
  --temperature 0.1
```

### Обучение ML моделей

```bash
uv run python scripts/train_ml_models.py --symbol CSI300
uv run python scripts/train_ml_models.py --symbol QQQ
uv run python scripts/train_ml_models.py --all  # Обучение на всех рынках
```

### Оценка и визуализация

```bash
# Генерация оценочных графиков из существующих результатов
uv run python main.py evaluate

# Расширенная академическая оценка (MAE/MFE, сегментация рыночных состояний, SHAP-атрибуция)
uv run python main.py evaluate-advanced \
  --output-dir docs/figures \
  --vix-file data/raw/vix_2015_2024.csv
```

### Переменные окружения


## Ключевые принципы проектирования

### 1. Отсутствие заглядывания в будущее (No Look-ahead Bias)

Инженерия признаков использует `shift()` для обеспечения работы только с историческими данными; выравнивание сигналов использует прямое заполнение `ffill`.

### 2. Независимые каналы (No Fusion)

Каждая модель независимо генерирует сигналы, конвертирует позиции и проводит бэктестинг. Отсутствие слияния сигналов обеспечивает честное сравнение.

### 3. Автономное кэширование LLM

Вызовы LLM API дороги и нестабильны; проект поддерживает автономное кэширование в формате `.jsonl` с возможностью возобновления.

### 4. Оптимизация для Apple Silicon

Модели LSTM на PyTorch автоматически обнаруживают и используют ускорение MPS (Metal Performance Shaders).

### 5. Обучение Walk-Forward

ML-модели используют механизм проверки Walk-Forward с окном обучения 912 дней и частотой повторного обучения 182 дня. Отбор признаков снижает количество исходных признаков с 55 до 17 ключевых через порог дисперсии и исключение коррелированных.

## Ключевые эмпирические результаты

| Результат | Детали |
|-----------|--------|
| **ML доминирует в акциях Китая** | LR: +9.34% vs BuyHold -5.23%, Sharpe 0.172, оборот 0.0019 |
| **LLM конкурентоспособен в акциях США** | Qwen 3.5-397B: +22.80%, рынок QQQ с Beta Drift (+67.16%) |

## Ключевые исследовательские вопросы

1. **Обучение vs Рассуждение**: Какая парадигма更适合 для количественной торговли — традиционное ML или LLM?
2. **Скорость vs Интеллект**: Низкозадержечное обучение или высокозадержечное рассуждение — что лучше?
3. **Облако vs Локально**: Стоит ли семантическое понимание LLM дополнительных затрат на API?
4. **Масштаб модели**: Как LLM разного размера влияют на задачи количественной торговли?
5. **События «чёрного лебедя»: Способно ли семантическое понимание LLM обеспечить защиту хвостовых рисков при экстремальных рыночных условиях?
6. **Кросс-рыночная генерализация**: Может ли одна система работать как с акциями Китая, так и с акциями США?
