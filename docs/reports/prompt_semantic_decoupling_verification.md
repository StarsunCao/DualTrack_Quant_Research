# Prompt语义解耦重构验证报告

## 1. 问题背景

### 发现的核心问题：语义过载（Semantic Overload）

之前的Prompt使用同一个词"sell"配合不同置信度阈值来区分"平仓"和"做空"：
- "sell" with confidence > 0.7 → CLOSE longs
- "sell" with confidence > 0.85 → OPEN short

**问题根源**：LLM对数值阈值感知迟钝，但对词义极其敏感。当"sell"带有做空暗示时，LLM会因新闻恐慌频繁打出高分，导致过度做空。

### 优化前回测结果（计划中引用）

| 策略 | 做空天数 | 总收益率 | 最大回撤 |
|------|---------|---------|---------|
| 全做空 | 551 (91.8%) | -21.42% | 34.71% |
| 阈值做空(85%) | 196 (32.7%) | -4.54% | 25.11% |
| 禁止做空 | 0 | +19.29% | 5.77% |

---

## 2. 语义解耦重构方案

### 核心改进：废除"sell"，引入"neutral"和"short"

| 原信号 | 新信号 | 含义 | 触发条件 |
|--------|--------|------|---------|
| sell | **neutral** | 清仓/持有现金（防御） | 不确定性、利润兑现、VIX升高 |
| - | **short** | 主动做空（极端看空） | 系统性风险、技术破位、流动性枯竭 |
| buy | buy | 开多仓 | 技术面+消息面共振 |
| hold | (移除) | - | 被neutral替代 |

### 信号映射逻辑

```python
if signal_lower == "buy":
    return confidence  # BUY: 正权重，按置信度建多仓
elif signal_lower == "neutral":
    return 0.0  # NEUTRAL: 清仓/持有现金，防御性操作
elif signal_lower == "short":
    return -confidence  # SHORT: 主动做空（仅美股有效）
elif signal_lower == "sell":
    return 0.0  # SELL: 向后兼容，映射为清仓（不再触发做空）
else:
    return None  # HOLD: 不产生目标仓位，保持现状
```

## 4. 效果分析

### 关键改进

1. **消除过度做空**：sell信号现在映射为清仓（weight=0），不再触发做空操作
2. **大幅降低回撤**：最大回撤从25-34%降至3.80%
3. **收益转正**：从亏损-21%/-4.5%转为盈利+10.58%
4. **保持向后兼容**：旧缓存中的sell/hold信号仍能正确处理

### 后续优化建议

1. **重新生成缓存**：使用新的Prompt（buy/neutral/short）调用LLM API生成新信号
2. **验证neutral信号**：新的neutral信号将更明确地表示"风险规避"
3. **验证short信号**：新的short信号将更加稀缺，仅在极端情况下触发
4. **VIX数据集成**：确保Prompt中提到的VIX判断条件在数据中实际存在
